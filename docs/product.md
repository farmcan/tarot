# MiaoTarot 产品与验证方向

更新时间：2026-07-20

这份文档只保留仍会影响产品决策的结论。历史调研、阶段性计划和已完成的 checkpoint 不再各自维护；当实现或真实用户证据改变结论时，直接更新这里。

## 产品假设

MiaoTarot 不是“用 AI 预测命运”，而是一个以标准塔罗为结构、以猫咪状态为情绪接口的自我观察工具：

> 抽一张猫咪塔罗，换个角度看清问题。

用户想要的不是更多神秘知识，而是一次低压力、能亲手参与、能准确描述当下感受的短体验。猫梗负责让人迅速产生“这就是我”的共鸣；牌义、牌阵位置和行动建议负责让结果不止是笑话。

当前需要用真实流量比较两个定位，而不是继续凭内部偏好选择：

- **猫状态翻译器**：先呈现可认领、可分享的猫咪 archetype，传统牌义在详情层。
- **可信的猫咪塔罗**：先呈现标准牌义与牌阵解释，猫咪视觉负责降低门槛。

## 体验原则

完整路径是：发现 → 第一次抽牌 → 翻牌高潮 → 理解与行动 → 分享或保存 → 在自然周期内返回。

1. **先给价值，再给设置。** 首次默认单张，只展示问题与开始按钮；牌数、内容包、逆位和音效属于更多设置。
2. **保留亲手选择的仪式感。** 洗牌时确定牌面与正逆位，用户选择的牌就是结果，不发生隐藏的二次抽牌。
3. **结果只出现一次。** 最后一张翻开后，一屏内优先展示核心猫牌、一句共鸣、一个小动作，以及分享、再来一次、继续深挖。
4. **分享默认保护隐私。** 默认不带用户问题；分享结果应能表达状态，又不剧透接收者自己的体验。
5. **复访匹配自然需求。** 先验证用户是每日、每周还是事件触发使用，再决定提醒、周报或订阅；不用 streak 制造焦虑。
6. **移动端是主体验。** 重点检查窄屏、现代手机和低高度横屏下的滚动、触摸目标、遮挡、翻牌与分享。
7. **保持用户能动性。** 文案提供一种解释和可执行的小动作，不给唯一答案，不制造恐惧或宿命感。

## 当前证据

仓库能证明产品已经形成完整闭环：22 张兼容牌组和 78 张完整牌组、交互式选牌与翻牌、本地历史、今日牌、分享图片与链接、可选 AI 解读、移动端 E2E 和 Cloudflare 发布链路。

仓库不能证明有人愿意持续使用或付费。当前没有已验证的 canonical 生产数据；以下仍是产品推断：

| 问题 | 当前判断 | 主要风险 |
| --- | --- | --- |
| 第一次点击 | 猫 + 塔罗 + 完整视觉有吸引力 | 首屏解释与选择仍可能过多 |
| 完成一次 | 仪式感和闭环足够 | 默认多牌动作多、移动端结果过长 |
| 分享 | 工程能力完整 | 分享入口远、身份表达和隐私不足 |
| 返回 | 有今日牌与本地历史原语 | 没有连续价值和真实 cohort |
| 付费 | 尚无证据 | 没有商品、价格、支付或长期价值验证 |

产品事件通过 Workers Analytics Engine 记录哈希后的匿名浏览器、标签页 session 和 reading id；不上传问题、笔记或牌面内容。它可以测量重复阅读，但在加入首访来源、曝光事件和 share token 前，还不能形成完整漏斗或分享回流归因。

## 指标与实验

候选 North Star 是 **7 天内完成第二次有价值阅读的匿名用户数**，而不是访问量或抽牌总数。确认自然使用频率后再调整窗口。

优先漏斗：

```text
landing → hero CTA → reading started → cards selected → cards placed
→ final flip → result CTA → share opened → referred reading
```

下一轮验证：

1. 发布唯一生产网址，并验证 D1 counter、Analytics Engine 和生产 smoke。
2. 为 30–50 名猫梗用户、塔罗用户和轻度怀疑者做无引导任务测试。
3. A/B 测试“猫状态翻译器”和“可信猫咪塔罗”，同时看完成、分享和 7 日二次阅读，不能只看 CTA 点击。
4. 分享默认隐藏问题，并用 share token 测量分享访问 → 完成 → 再分享。
5. 在基础结果后测试透明的付费意图：一次深读、主题报告或数字牌组；功能未开放时明确说明，不伪造结账。

以下阈值是实验前的团队决策线，不是行业基准：

- 开始后完成率低于 60%：先修路径，不做付费。
- 看到结果 CTA 的用户分享意图低于 8%：先修结果内容和隐私。
- 7 天二次阅读低于 10%：不做订阅。
- 付费详情点击低于 5%：先换价值主张，不接支付。
- 至少出现 10 个真实小额购买或可退款预售，并有重复使用，再讨论订阅。

## 路线与优先级

### P0：先建立真实使用证据

- 默认单张并收起高级设置。
- 合并重复结果，缩短最后翻牌到行动入口的距离。
- 分享默认隐藏问题，补首访来源、曝光事件和 share token。
- 全量简体中文与 78 张内容编辑 QA。
- 修复低高度横屏布局，并完成手机真实路径测试。

### P1：有初步流量后

- 比较两个定位和对应分享卡。
- 把今日牌明确为“共同今日主题”，加入历史笔记、行动反馈和 7 天回看。
- 测试一次深读、主题报告/数字牌组、B2B 主题页三类真实 offer。
- 只为有明确人群与分享句的主题增加独立入口。

### P2：证据支持后才做

- 跨设备账户、订阅和自动周/月报告。
- 课程、社区、实体牌量产。
- 真人顾问市场。

### 延后假设：创作者牌组平台

长期可以把 MiaoTarot 从单一自有牌组扩展为邀请制创作者牌组平台：创作者提供画面、角色设定与受众，平台提供标准塔罗结构、抽牌与解读体验、商品展示、支付和分成。这个方向当前只作为待验证假设记录，不代表已经决定建设开放社区、自助上传、评论、关注、审核或自动分账系统，也不需要现在确定具体分成比例。

最重要的前置风险是双边平台冷启动：

```text
没有用户 → 创作者不愿投入 78 张
没有好牌组 → 用户没有理由来
```

因此第一步不是招募创作者或开发平台能力，而是先验证 MiaoTarot 能否获得并留住真实用户。用户量应以真实访问、完成阅读、分享回流和重复使用判断，不以功能数量、内部体验或一次性曝光代替。只有核心产品形成持续用户量后，才考虑从少量邀请制、人工上架和人工结算的创作者牌组试点开始；未通过试点前不建设开放社区。

这也是项目级终止条件：如果完成首版上线、核心路径修复和有边界的获客/复访实验后，仍无法形成真实且持续的用户量，就停止创作者平台、付费和新增功能投入，直接废弃 MiaoTarot 项目，不因已经投入的时间或牌面资产继续维护。

## 收入假设

当前最匹配的消费者路径是：

```text
免费完整抽牌与基础解释
→ 针对具体问题的一次深读或 credits
→ 历史与回看产生真实复访
→ 再测试订阅
```

可以并行验证的早期收入是品牌/创作者定制主题页，以及限定数字或实体牌预售。它们利用现有主题引擎和视觉资产，不要求先拥有大规模日活。

不要把“更多普通 AI 文本”当作订阅价值。相邻产品的共同证据是：用户更愿意为具体问题的可信回应、跨时间累积价值、学习与身份、收藏所有权或真人关系付费。

## 信任边界

- 明示用于自我观察与娱乐，不替代医疗、法律、财务或危机支持。
- AI 只解释已经抽出的牌，不能重抽、发明牌或给固定预测。
- 抽牌、历史和基础结果不依赖 AI；provider key 只存在服务端。
- 不上传用户问题、笔记或牌面内容到产品分析。
- 不使用恐吓、诅咒、复合诱饵、模糊试用、自动续费或假稀缺。
- 知名猫梗只有在商业改编许可明确后才能进入生产资产。

## 研究依据

这里仅保留会影响当前决策的证据入口：

- [Zappallas FY2025 财报](https://www.zappallas.com/wp-content/uploads/2025/06/tanshin_2504_4Q_250613.pdf)：轻量内容、深度内容、真人服务和 B2B 可以分层经营。
- [Labyrinthos App Store 说明](https://apps.apple.com/gb/app/labyrinthos-tarot-reading/id1155180220)：大量免费用户和完整功能不自动产生健康付费。
- [NYT 2025 10-K](https://www.sec.gov/Archives/edgar/data/71691/000007169126000011/nyt-20251231.htm)：有限、共同节奏的免费内容可以成为更大产品的入口。
- [RevenueCat 2026 subscription benchmarks](https://www.revenuecat.com/blog/growth/subscription-app-trends-benchmarks-2026)：持续价值决定长期结果，偶发需求更适合按次或混合变现。
- [CHI 2026 AI Tarot research](https://arxiv.org/abs/2602.11367)：AI 解读应保留模糊性、替代视角和用户能动性。
- [Fogg Behavior Model](https://www.behaviormodel.org/home)：首次路径应同时降低能力门槛并给出明确 prompt。
- [Reforge ICED theory](https://www.reforge.com/blog/iced-theory-growing-infrequent-products)：复访节奏应匹配自然需求频率。
- [Superhuman PMF method](https://review.firstround.com/how-superhuman-built-an-engine-to-find-product-market-fit/)：只在用户实际重复使用后调查不可替代价值。

这些材料证明相邻需求和经营模式存在，不证明 MiaoTarot 已经具备产品市场匹配；最终以自己的行为、访谈和支付数据为准。
